Qtopia Sync Agent changes for 4.3.1

Overview

This page describes changes to Qtopia Sync Agent 4.3.1.

  • Fixed some data corruption bugs and implemented test cases for the Qtopia side of the connection.
  • Documentation has been significantly updated.
  • Corrected issues that impacted building Qtopia Sync Agent from source code.
  • The connection timeout code was removed due to problems it was causing with the Greenphone.

Known issues

  • Appointments starting and ending at midnight are mis-reported as all day events.
  • An obvious crash bug somehow made it through our QA process. Contact support-embedded@trolltech.com if you require the patch to correct this.
  • Records are being duplicated when synchronizing multiple times.
  • Tasks does not sync due to an error handling the low-level protocol.
  • Objects of an unexpected type returned from Outlook can cause problems. For example, a distribution list in the Contacts folder is not ignored and is accessed like a Contact, potentially causing problems.
  • Items created in Outlook are duplicated at every sync.
  • Tasks modified on the server are not changed on the device.
  • Appointments with no alarms are not read correctly (Repeat, Notes, Categories and Custom Fields from Outlook are not seen by Qtopia).
  • Certain characters appearing in text (eg. the elipsis character) can cause Qtopia Sync Agent to crash due to a text encoding bug.
  • Long notes are not transferred in full (the beginning part of the note is cut off).
  • The clocks on the desktop and device must match or some items may not be synchronized.
  • All day events do not synchronize correctly between Qtopia and Outlook.
  • Outlook items that are not known to Qtopia Sync Agent but have a last modified timestamp that is before the sync timestamp are not transferred. This can be observed by importing items from a .pst file (they may not sync).
  • Changing a recurring event in Qtopia to a non-recurring event causes Qtopia Sync Agent to crash.
  • Birthdays and anniversaries are duplicated in the calendar.
