DialUp Plugin Specification

Allows serial port and IrDA modem network connections.

Adds two service types in the Network Settings - one for serial ports such as CF modem cards, the other for IrDA connections.

Status: RT

Use Cases
  • CF modem

    User inserts CF modem, selects the dialup connection, and uses the internet.

  • IrDA via cellphone

    User places an IrDA phone within range, selects the dialup connection, and uses the internet.

  • Demand dialing

    User inserts CF modem or places an IrDA phone within range, and use the internet. The modem is dialed automatically (user has earlier enabled that option).

  • ISP configuration

    When user subscribes to an ISP, he receives a disk containing setup software, including a package for Qtopia, which includes phone number and other configuration details. User installs the package and then has dialup access without manual configuration.

Features
  • Parameterized: most parameters for the ppp connection can be set by defaults in a configuration file, so that, for example, an ISP can provide a complete service definition that the user just enables and uses - with the phone number, password, etc., predefined.
  • Serial modem: when a CF modem is inserted, the dialup services become available (see Network Monitor applet).
  • IrDA modem: available when the user has enabled IR (not implemented: and a phone is discovered).

Prerequisites
  • IrDA discovery: the irda service (/etc/rc.d/init.d/irda) must run irattach in discovery mode (-s option).
