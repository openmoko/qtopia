Use Case Tests

What do I do now

  1. Using Button Editor, ensure that "Add new task" is bound to the Hold action of an appropriate key (some devices have a Todo key, otherwise take another key).
  2. Start with empty Todo list.
  3. While running other application, for Table 1 data 1..6, activate the "Add new task" key, and:
    1. Enter information, press OK.
    2. Use clock applet to advance time by some amount to fake waiting.
  4. Run Todo List application.
  5. Observer that all tasks are in the order of importance (budget, car, tax, raise, hat, reports).
  6. Check the left-hand checkbox for the Fix Car item; check it moves to the end of the list.
  7. Click the priority field of File Progress Reports, change priority to 2; check it moves to the fourth position.

Different hats

  1. Enter all Table 1 data.
  2. Select Person category from category filter combobox.
  3. Check that only items 1, 2, 3, 4, and 7 are shown.
  4. Select Kids category from category filter combobox.
  5. Check that only item 7 is shown.

Feature Tests

Add Task

  1. Click .
  2. Enter details, non-empty description.
  3. Click OK.
  4. Check that item is added.

Add Task - Empty

  1. Click .
  2. Enter details, but no description.
  3. Click OK.
  4. Check that item is not added.

Delete Task

  1. Click .
  2. Check that prompt is correct.
  3. Click OK.
  4. Check that item is gone.

Completion

  1. Click on empty checkbox to left of item.
  2. Check that item is now recorded as completed.
  3. Click on checked checkbox to left of item.
  4. Check that item is now recorded as not completed.

Priority

  1. Click on priority number towards left of item.
  2. Choose a priority.
  3. Check that item is now recorded as having chosen priority.

Categories

  1. Optionally, click on a task.
  2. Click on .
  3. Choose a category from the combobox.
  4. Click OK.
  5. Check that item is now recorded as having chosen category.

Categories - Multiple

  1. Optionally, click on a task.
  2. Click on .
  3. Choose "..." to right of category combobox.
  4. Choose multiple categories by clicking checkboxes.
  5. Click OK.
  6. Check that item is now recorded as having chosen categories, showing "<first category> (Multi.)" in combobox.

Due Date

  1. Check that for tasks with the same completedness and priority, those with a Due Date precedes tasks without a Due Date, and that of those with a due date, tasks with the earlier due dates precedes tasks with later due dates.

Today Plugin

  1. Check that changes are reflected immediately in the Today application.

Beam

  1. Check beaming to other Qtopia device.

Beam - Palm OS

  1. Check beaming from Palm OS device.
  2. Check beaming to Palm OS device.

Beam - Pocket PC

  1. Check beaming from Pocket PC device.
  2. Check beaming to Pocket PC device.

Find

  1. Click find
  2. Typing text should immediately select match, or show "Find: not found" in status (tool) bar.

Filter by category

  1. Check that filtering works for tasks in no category: should only be visible in All and Unfiled categories.
  2. Check that filtering works for tasks in one category: should only be visible in that one category, and All.
  3. Check that filtering works for tasks in more than one category: should only be visible in those categories, and All.

Filter by completion

  1. Only specified events should be visible.
