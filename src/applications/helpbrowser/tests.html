Qtopia/Help Browser

Test cases 



 Qtopia help index

Test Purpose
 Verify that the Help Browser starts with the Qtopia help index.
Test Description
  1. Verify that the help browser starts with the help home page showing the Qt logo and an index to the most important topics.
  2. Browse a bit through the documentation by clicking on some links.
  3. Click on the "Home" button.
  4. Verify that the help returns to the index page.
Expected TestResult
 The Home page should show an index with the following items:
  • Using Qtopia
  • Synchronizing with a PC
  • Applications
  • Games
  • Settings
  • Documents
  • About Qtopia
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test


 Navigating

Test Purpose
 Verify that you can navigate through the help
Test Description
 For this test it is important to start with an empty history for the navigate buttons. So close and reopen the Help browser if neccessery.
  1. Verify that both navigate buttons (Left arrow and Right arrow) are grayed.
  2. Click "Applications"
  3. Verify that the "Left" arrow gets enabled.
  4. Click "Calculator"
  5. Click "Simple mode"
  6. Verify that the current help page gives a description of the Simple Mode of the calculator.
  7. Click on the "Left" button and verify that this brings you back to the "Calculator" help.
  8. Verify that the "Right" arrow is now enabled too.
  9. Click on the "Left" button again and verify that this brings you back to the "Applications" help.
  10. Click two times on the "Right" button and verify that you are back in the "Simple Mode" help page.
  11. Verify that the "Right" button is disabled again.
  12. Click on the "Home" button
  13. Click in the "Left" button multiple times and verify that this 'replays' exactly the sequence in which you have browsed through the information.
  14. Verify that after 4 clicks you are back in the Home page again and the "Left" button is disabled.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test


 Adding a Bookmark

Test Purpose
 Verify that a bookmark can be set for quick acces to help
Test Description
  1. Click "Home" if you are not in the home page
  2. Click "Settings"
  3. Click "Backups"
  4. In the Bookmark menu click on "Add Bookmark"
  5. Click on the Bookmark menu and verify that "Backups" is now available in the list of bookmarks.
  6. Click on the "Home" button and then on "Backups" in the Bookmark menu.
  7. Verify that the backups help page is shown again.
  8. Repeat the same but start from an arbitrary page somewhere in the available help.
  9. Close and reopen the Help Browser and verify that the same bookmarks are still available.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test


 Removing a bookmark

Test Purpose
 Verify that a bookmark can be removed from the bookmark list
Test Description
  1. Create a bookmark for the "Contacts" and "Calendar" applications.
  2. Click the "Bookmark"  menu and select the "Contacts" bookmark from the list.
  3. Verify that the browser shows the "Contacts" help page.
  4. Click the "Bookmark" menu and select "Remove from bookmarks"
  5. Open the "Bookmark" menu again and verify that the "Contacts" bookmark is removed from the list.
  6. Click the "Bookmark" menu and select "Remove from bookmarks" again
  7. Open the "Bookmark" menu again and verify that nothing is changed in the list, i.e. at least the "Calendar" bookmark should still be available.
  8. Click the "Bookmark"  menu and select the "Calendar" bookmark from the list.
  9. Verify that the browser shows the "Calendar" help page.
  10. Click the "Bookmark" menu and select "Remove from bookmarks"
  11. Verify that the "Calendar" bookmark is removed from the menu.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test
