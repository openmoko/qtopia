Qtopia/Date and Time

Test cases 



 Time adjustment

Test Purpose
 Verify that the time can be adjusted.
Test Description
  1. Set the Time format to 24 hour.
  2. Verify that the Hour can be changed between 0 and 23.
  3. Change the Hour and Minute using the up/down buttons.
  4. Also try changing the time using the keyboard.
  5. Select a new time and tap OK 
  6. Verify that the new time is visible in the clock applet in the taskbar.
  7. Open Date/Time again, select a different Timezone and tap OK.
  8. Verify that the time in the clock applet has changed to match that of the timezone.
  9. Set the Time format to 12 hours.
  10. Verify that the Hour can be changed between 1 and 12.
  11. Set a time + AM and tap OK.
  12. Verify that the time is shown on the clock applet on the taskbar.
  13. Set a time + PM and tap OK.
  14. Verify that the time is shown on the clock applet on the taskbar.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test


 Select from six timezones

Test Purpose
 Verify that you can set the timezone to one of the six available timezones
Test Description
  1. Tap the World Time icon.
  2. Tap on one of the six city buttons, then tap and hold the stylus around the world map. Different cities are displayed with their local time. Release on any city. 
  3. Verify that the city button now has the selected cities' time in it.
  4. Tap Close (x) to close the World Time viewer.
  5. Tap the Time Zone selector. 
  6. Verify that the timezone previously selected from theWorld Time application is now in the list.
  7. Select the newly added timezone and tap OK 
  8. Verify that the time in the clock applet has changed to match that of the timezone.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test


 Time format

Test Purpose
 Verify that the time presentation can be configured.
Test Description
  1. Start the Clock, Calendar and Today applications and keep them running during the test.
  2. If required create an event in Calendar for today (the event should be visible in Calendar and in Today).
  3. Set the Time Format listview to 24 hour mode. 
  4. Verify that the AM/PM listview is disabled.
  5. Verify in Clock that the displayed time is shown in 24 hour format.
  6. Verify that the event(s) are shown in 24 hour format in Today.
  7. Create a new event in Calendar and verify that the time edits in the dialog show the time in 24 hour format.
  8. Verify that the time presented in the clock applet on the status bar is shown in 24 hour format.
  9. Set the Time Format listview to 12 hour mode. 
  10. Verify that the AM/PM listview is enabled.
  11. Verify in Clock that the displayed time is shown in 12 hour format.
  12. Verify that the event(s) are shown in 12 hour format in Today.
  13. Create a new event in Calendar and verify that the time edits in the dialog show the time in 12 hour format.
  14. Verify that the time presented in the clock applet on the status bar is shown in 12 hour format.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test

 Configuring date format

Test Purpose
 Verify that the date format can be configured
Test Description
  1. Open the Clock and Calendar applications and keep them running during the test.
  2. In the Date/Time app select a date format.
  3. In the Clock app verify that the presented date at the bottom of the screen is in the expected format.
  4. In the Calendar app verify that the presented date in the date picker is in the expected format.
  5. Repeat the same test for all the possible date formats.
Expected TestResult
 The following output is to be expected for the different formats (assuming the date is Wednesday Feb 19, 2003):
  • M/D/Y: Wednesday Februari 19, 2003
  • D.M.Y: Wednesday, 19.Februari 2003
  • Y-M-D: Wednesday, 2003, Februari 19
  • D/M/Y: Wednesday, 19 Februari 2003
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test


 Setting a date

Test Purpose
 Verify that the current date can be selected
Test Description
  1. Tap on the date button and select a date from the date picker dialog (please play a bit with all the controls before you select a final date).
  2. Verify that after you tapped on the date the picker dialog closes automatically and the selected date is shown on the date button.
  3. Tap OK.
  4. Open the Clock app and verify that the selected date is shown at the bottom.
  5. Open Date/Time again and set the date to 19 Februari 2003, set the timezone to Brisbane, adjust the time to 06:00 and tap OK.
  6. Verify in Clock that the presented date is 19 Februari 2003.
  7. Open Date/Time again, set the timezone to Oslo and tap OK.
  8. Verify in Clock that the presented date is18 Februari 2003 and the time is approx 21:00.
  9. Open Date/Time again, set the timezone to Los Angeles and tap OK.
  10. Verify in Clock that the presented date is18 Februari 2003 and the time is approx 12:00.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test
