 Adding a service using DHCP


Test Purpose
 Verify that a network service can be add and configured for DHCP
Test Description
  1. Insert an Ethernet LAN card in the CF slot and hook up to a network.
  2. Go to the services tab and remove all services (if any available).
  3. Tap the "Add" button.
  4. Select LAN -TCP/IP
  5. Tap "Add" again
  6. Verify that a LAN account has been created.
  7. Enter a new name, e.g. LAN-TEST
  8. Verify that DHCP has been selected by default.
  9. Verify that the IP tab is disabled.
  10. Tap "OK".
  11. Verify that the list of available services now reports a "LAN-TEST".
  12. Go to the Current tab.
  13. Verify that a IP address is mentioned in your network domain.
  14. Start Terminal in the applications tab (if available) and verify that you can ping to a host that should be accessible.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test


 Adding a service for manual IP

Test Purpose
 Verify that a network service can be add and configured for manual IP 
Test Description
  1. Insert an Ethernet LAN card in the CF slot and hook up to a network.
  2. Go to the services tab and remove all services (if any available).
  3. Tap the "Add" button.
  4. Select LAN -TCP/IP
  5. Tap "Add" again
  6. Verify that a LAN account has been created.
  7. Enter a new name, e.g. LAN-TEST
  8. Select "Configure IP manually".
  9. Verify that the IP tab gets enabled.
  10. Verify that the IP address field is empty and tap OK.
  11. Verify that the list of available services reports a "LAN-TEST".
  12. Go to the Current tab and verify that the IP address shows "eth0".
  13. Go to the Services tab again.
  14. Select the LAN-TEST account and tap "Properties"
  15. Go to the IP tab and enter valid settings for the IP address, Broadcast, Gateway and DNS.
  16. Tap "OK".
  17. Go to the Current tab.
  18. Verify that a IP address is mentioned in your network domain.
  19. Start Terminal in the applications tab (if available) and verify that you can ping to a host that should be accessible.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test


 Dial up modem

Test Purpose
 Verify that a network service can be installed for a dial up modem.
Test Description
  1. Insert a dial up modem card into the CF slot.
  2. Go to the services tab of the Network application.
  3. tap "Add" and select "Dial-up - PPP" + "Add" again.
  4. Enter valid information in the Username, Password and Phone fields.
  5. Verify that the settings in the Modem, Network and Proxies tab are according to the ISP settings.
  6. Select "Manual connect and disconnect" in the Modem tab and tap OK.
  7. Close the network application.
  8. Verify that a PPP connect icon is shown in the task bar with a red cross (no connection).
  9. Connect the modem to a phone line.
  10. Tap on the PPP connect icon and tap "Connect".
  11. Verify that the plugin shows the progress of the connection and finally shows connected.
  12. Start Terminal in the applications tab (if available) and verify that you can ping to a host that should be accessible.
Expected TestResult
 All the verification steps as described in the Test Description must be met.
Testnorm/Criteria
 Exact Match
Test Suitability
 Manual test
