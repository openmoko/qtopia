Communications: Presence
Functional Description

Presence in Qtopia refers to SIP presence and is currently only relevant for Qtopia's VoIP.

Using the VoIP Settings program it is possible for a user to modify their own presence state and the presence state of others is shown in the Address Book application in the contact details page.

Features

  • Presence API

Presence API

NOT YET RELEASED

The presence information allows the user to identify whether the contacts are availabe for communications at a glance.
This encourages successful communications while avoiding unanswered calls etc.

Standard compilance
The presence API abtracts the industry standards in presence
to provide a generic interface between applications and presence servers.
SIP and XMPP(Jabber) protocols are benchmarked to create the initial interface.

Presence Data
The presence API supports contacts to have multiple resouces. i.e. A contact can have separate presence status for IM and Voice call.
The priority based resource look up is supported.

Basic Presence Information
The basic presence status of a presence entity can be described in Online, Offline and Away status.
This status is determined by compiling all the status on available resources.

Rich Presence Information
The presence API supports extended presence elements such as mood, activities, location etc

Presence Information and User Interface
A presence status icon describing Online/Offline/Away appears whenever contact's name is shown.
The status line appears if the screen realestate allows.
A notification is shown on the homescreen whenever there is a status change to contacts displaying the new status and status line.
