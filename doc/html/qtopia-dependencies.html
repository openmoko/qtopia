Introduction

In order for Qtopia to run as intended the following prerequisites must be satisfied:

  • a suitable compiler must be installed
  • Linux kernel that provides shared memory, mmap, and sockets.
  • If Qtopia is being built with Safe Execution Environment - (SXE) support, then the kernel must be patched to have LIDS support.
  • If Qtopia is being built with Digital Rights Management(DRM) support, then version 2.4 of drmagent from BeepScience must be installed.

    Note: A commercial license from BeepScience is required to use the BeepScience drmagent.

  • If Qtopia is being built with Helix please refer to Helix for further information about using Helix with Qtopia.
  • Linux with Framebuffer support.
  • For dependencies related to playing sounds see Sounds and Qtopia
  • For dependencies related to Bluetooth see Bluetooth Support

Sounds and Qtopia

Qtopia can be configured to play sounds via a media engine integration or by playing sounds directly to the sound hardware. One or more media engine integration can be enable via the -mediaengines option see also

    ./configure -help -verbose

For details on how to use the Helix Integration see Qtopia Helix Integration.

GStreamer Media Engine Integration

Before attempting to configure Qtopia with GStreamer support enabled ensure that all requirements as stated at http://gstreamer.freedesktop.org/ are met.

Directly Playing Sounds to Hardware

When a media engine integration is not used Qtopia requires that either:

  1. A OSS Linux sound driver is used
  2. A ALSA sound support with OSS compatibility

In addition Qtopia requires that /dev/dsp is writable and supports the following ioctls:

  • SNDCTL_DSP_SETFRAGMENT - Qtopia will set this to 0x4000c.
  • SNDCTL_DSP_SETFM - Qtopia will set this to AFMT_S16_LE
  • SNDCTL_DSP_STEREO - Qtopia will set this to 1/true.
  • SNDCTL_DSP_SPEED - Qtopia will set this to 44100.
  • SNDCTL_DSP_GETOSPACE.

Qtopia also requires that /dev/dsp can be opened as both blocking and non-blocking. The values set above are Qtopia's preferred settings for the audio device, and some deviation is allowed. However smooth playing of audio is not guaranteed if the above ioctls cannot be set to the Qtopia preferred settings.

For example your device only plays at 22500 speed, then it is required to report this when the SNDCTL_DSP_SPEED ioctl is called and should not result in a drop in audio quality. However not supporting GETOSPACE, non-blocking write or SNDCTL_DSP_SET_FRAGMENT will almost certainly result loss of audio quality.

Video4Linux

The Camera application requires a kernel with Video4Linux version 1 or version 2 support. This is not provided with Qtopia.

Please refer to the official Video for Linux resources site for details about this API.
