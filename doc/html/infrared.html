Introduction

Qt Extended provides library support for infrared development, as well as infrared file transfer implementations for end users.

The Qt Extended infrared support relies on Linux-IrDA, an implementation of the IrDA standards that has been included with the Linux kernel since the Linux 2.2.x series. Infrared hardware support is dependent on Linux-IrDA; Qt Extended does not provide direct support for any hardware.

Qt Extended Configuration

Infrared support is included by default in Qt Extended. No additional configuration options are required.

Functionality

Local device control & remote device discovery

The local infrared device can be turned on/off through the QIrLocalDevice class. This class can also be used to find remote infrared devices. Alternatively the QIrRemoteDeviceWatcher class can be used to be notified when a particular remote device is found.

Access to LM-IAS (Link Management Information Access Service)

The LM-IAS database contains the details that are advertised to remote infrared devices that are searching for particular services. The QIrIasDatabase class allows these service attributes to be added and removed from the local LM-IAS database.

Tiny TP socket support

Tiny TP (Tiny Transport Protocol) is an IrDA protocol that provides flow control on top of the IrLMP (Infrared Link Management Protocol) layer. Qt Extended provides support for Tiny TP client and server sockets through the QIrSocket and QIrServer classes. For convenience, services published through QIrServer are automatically added to the local LM-IAS database.

Object Push Profile support

Qt Extended provides support for both client and server implementations of the Object Push Profile, a simple file transfer protocol built on top of the IrOBEX protocol. This includes specific support for vCards and vCalendar transfers as well as the transfer of arbitrary files.

Object Push clients can be implemented by using QObexPushClient together with QIrSocket. Object Push servers can be implemented by using QObexPushService together with QIrServer. See the QObexPushClient and QObexPushService class documentation for more details.

Qt Extended has built-in client and server implementations of the Object Push Profile. End users can send and receive files over infrared; file transfer progress indicators are displayed to the user, and any received vCards can be saved to the address book.
