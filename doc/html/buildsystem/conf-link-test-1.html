link_test

Usage

    CONFIG+=link_test

Description

Opt in to the automatic link test. The link test helps to ensure that undeclared dependencies do not exist. GCC does not complain when creating a shared object (library or plugin) with undeclared symbols and there is poor support for reporting these errors at runtime. By doing a link test, any undeclared symbols become errors.

Note that this is on by default for all Qtopia projects with type plugin so setting CONFIG+=link_test is only required for external projects. If present, no_link_test overrides link_test.

See also CONFIG Values and no_link_test.
