What is the build system?

The build system is responsible for building Qtopia as well as third party applications.

For definition purposes, the build system is made up of the following files

  • configure

    This is the initial entry-point into the build system. It is used to setup the build tree.

  • bin/qtopiamake

    This is called by configure and doubles as the the initial entry-point for external projects.

  • bin/*.pm

    These are support modules for the perl scripts that make up the build system.

  • Makefile

    This is the second entry-point into the build system. Makefiles are created by qtopiamake.

  • Makefile.target

    This is the file that qmake creates for each project. It should not be called directly.

  • *.pro

    This file is read by qmake when it makes Makefile.target. There is one of these files for each project.

  • *.prf

    These are features files. They can be loaded explicitly but are typically loaded implicitly.

  • *.pri

    These are miscelaneous build system files. They are included from .pro or .prf files.

  • src/build

    This is the home of most of the build system's files.
