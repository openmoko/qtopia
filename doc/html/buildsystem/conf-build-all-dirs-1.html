build_all_dirs

Usage

    # Limit to these two subdirectories. Leave SUBDIRS empty for all subdirectories.
    SUBDIRS=foo bar
    CONFIG+=build_all_dirs

Description

Used by a subdirs project to specify that all available subdirectories should be built rather than relying on the PROJECTS variable. This is superior to having nested subdirs projects because dependencies cannot be processed when that is done.

See also CONFIG Values, PROJECTS, Dependencies, and Project Trees.
