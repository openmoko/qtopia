Overview

The Qt Extended Theme Editor is a tool intended to help designers create themes for Qt Extended. The Theme Editor provides a graphical user interface designed to facilitate common tasks in creating themes for Qt Extended. The Theme Editor can create, load and save themes, and within the themes can populate themed views, edit color palettes and set icon images.

The theme editor contains several tools to assist in creating themes, accessible from the 'Tools' menu. These tools include an XML editor to directly modify the themed view XML files, a color scheme editor and selector to assist in creating color schemes for a theme, and a simulated Value Space for previewing theme elements which use values from the Value Space. Some of these tools open in their own windows, others dock inside the main window. For the latter type you can toggle their visibility from inside the 'Tools' menu.

Basic Tasks

Creating a New Theme

To create a new theme in the theme editor select 'New Theme' from the 'File' menu. This will present a dialog, requesting some required information about the theme you are creating.

A name is required for every theme, as themes are often referred to by name. It is best if this name is not already used by another theme. The name can be changed at any time in the properties dialog.

Height and width(in pixels) are recommended to be specified here. This is primarily used by the editor, and other people looking at the theme, as Qt Extended will use the device's screen size at all times. The size chosen here will be the size of each page in the editor. The height and width can be changed at any time in the properties dialog, however the theme will need to be reloaded for changes to take effect.

Many user interface components in Qt Extended are themeable, and a theme does not need to specify what all of them look like. The most common user interface components are available to choose from, and the selection here determines which user interface components the theme will specify an appearance for. User interface components can be added or removed at a later stage by editing the theme configuration file. It is not strictly required, but it is recommended that a theme specify the appearance of at least one user interface component.

Once you have decided upon these details press the 'Ok' button and an empty theme will be created, based on the provided information. All of the themed views will be empty at this stage, but the theme editor should make filling them fairly easy.

Opening an Existing Theme

Themes can be stored in one of two ways: either as a theme package or as a collection of files in a directory. You can save and load both of these formats. Note that only one theme may be opened in the editor at a time; the open actions will be unavailable if another theme is currently open. Select the 'Close' option from the 'File' menu if you wish to open another theme (but ensure you have saved any work you wish to keep first).

To open a theme package, select 'Open Package' from the 'File' menu, and then browse to the package you wish to open. Once it is selected, press the 'Open' button.

To open a theme stored in a directory select 'Open Source Directory' from the 'File' menu. The 'Open Last Directory' option will open the last source directory which you opened with the 'Open Source Directory' option, or saved to a directory. The directory is expected to conform to a certain structure, with one directory containing all of the files except possibly images (it is referred to in the dialog as the 'XML directory:' as .xml files are its primary content). The configuration file may be in the directory above the XML directory, and it must be named '<name>.conf' where <name> is the name of the theme. If the images for the theme are stored in a separate directory, tick the 'Separate Images Directory' check box and select the correct path for the directory which contains the images. When you click the 'Ok' button, the theme will be loaded. If the paths provided are not valid, an error will appear and no Theme will be loaded. Should the paths be valid, but a theme organized as above is not found, an empty theme will be loaded. It is recommended that the 'New Theme' option be used instead if you wish to create a new theme in a specified location.

Saving a Theme

Themes can be saved in the same formats as they can be loaded. To save a theme, a theme must first be open. If no theme is open, the save options will be disabled. If you wish to save a theme in this case, first open or create a new theme as detailed above.

To save a theme in a package, select 'Save Package' from the 'File' menu. Browse to where you wish to save it and enter an appropriate file name. If the file name you entered does not end in '.qpkg' it will be automatically appended to the file name when you save.

To save an entire theme into a directory select the 'Save All' option from the 'File' menu. This will prompt for the directory or directories which you would like to save it in. These do not have to be the same as the ones you loaded from, although that is what they default to. The structure of the directories is the same as for opening a theme, as detailed above.

If you wish to save only the current view, and you loaded the theme from a directory as opposed to from a package, select the 'Save' option from the file menu. This will only save the current view, and any resources (such as pictures) that it depends on. If you are concerned about losing your work, note that the Theme Editor auto saves the work frequently, and these auto saves will be offered for recovery if the theme editor exits unexpectedly (they will not overwrite files in the specified directory). If the theme editor exits in an expected manner, but has unsaved views, it will require confirmation before the work is lost.

Modifying Theme Items

Once you have a theme open you can add, remove, and modify items in it. If these methods are not suitable for your needs, an XML editor is built into the theme editor, and you can modify the items directly in it.

To add an item to the active view, press and hold the left mouse button on an item in the item palette, and drag it onto the page or the item browser. The highlighted item at the time you release the mouse button will be the parent of the newly created item. Basic items are initially empty, and so may not be visible until you add content to them.

To delete an item, simply select it and press the delete key on your keyboard. Remember that you can select items by clicking on them in the central view, or by clicking on their entry in the item browser.

To modify the positional properties of an item in the active view, you can click on it in the main area to select it, and then either drag it to a new location or drag the resize handles that have appeared on top of it so as to alter the item's size. When moving an item, the item which is highlighted in red below is the item which will be its parent item when it is moved there. If you wish to move an item without reparenting it, use the item editor's controls for altering x and y. If you wish to reparent an item without moving it drag the entry for the item you wish to reparent onto the item you wish to be the new parent. Note however that position is expressed relative to the parent item, and so this may result in an unintended change of position.

If you are having difficulty clicking on an item to select it, press the right mouse button anywhere on top of it and a menu will appear listing the items at that location. Selecting an item from the menu will select it in the main view.

If you are having difficulty clicking on an item to drag it into a new position, holding down shift while dragging the resize handles will cause the item to move instead of resizing. It does not matter which resize handle you drag.

Another method of modifying an item is through the item editor. The item editor displays the properties of the currently selected item, and these properties can be modified. The available properties will depend upon the type of item selected.

Other Tasks

Modifying Theme Meta-data

Some of the theme meta-data can be edited from the 'Properties' option in the 'Theme' menu. This allows convenient editing of commonly used meta data (this includes the name and default dimensions which you may have specified in theme creation). Should you need to alter other parts of the theme meta-data, the 'Edit .conf file' option in the 'Theme' menu provides convenient access to the configuration file of the theme. A small text editing window will be opened, where you can make the required changes and then save the file. Note that this will still work when opening the theme from a package.

Modifying the XML File

The theme editor does not do everything (see 'Limitations' below), and so editing of the XML files by hand may still be necessary. To assist this when needed, the theme editor contains an XML editor for directly editing the XML of the theme you are working on.

To access the XML Editor, select 'XML Editor' from the 'Tools' menu. The XML Editor will now appear. Note that you cannot edit the theme except via the XML Editor while it is open. The theme editor is still running, and will display the theme for which you currently have the .xml file open. This view will be updated when you save the XML file (by pressing the 'Write XML File' button), so that you can view your changes. You can switch to another XML file in the theme using the box at the top of the editor. Click the 'Close' button to return to the main view.

Value Space Integration

Themes often wish to interact with the Qt Extended Value Space and the Theme Editor attempts to facilitate this. The theme editor loads an example Value Space which contains all of the commonly used Value Space items. There are two views on the Value Space, a common view and a full view. The common view provides easy access to the most common Value Space items, and allows you to change their values directly, to witness the effect on the theme. The full view provides the full range of Value Space items which might be expected on a Qt Extended based phone, and also allows you to edit their values and view the effects on the theme. In both views, you can click and drag the Value Space item label to a text field to get the full Value Space path.

Color Scheme Editing

The theme editor assists both in the creation and selection of color schemes for use in a theme. From the color scheme palette, the available color schemes are show with options to create new or edit existing color schemes. The check box next to each color scheme indicates whether this color scheme should be listed as a possible color scheme for use with this theme; the user will only be able to select from the checked themes. If you select a theme, the palette of the views currently shown will be updated, so you can preview you theme with the selected color scheme.

Icon Selection

To conveniently replace some of the icons in Qt Extended the theme editor allows for easy replacement of the icons. Select the 'Icon Editor' option from the 'Tools' menu, and the Icon editor will load. In this window you can navigate the icons which are in Qt Extended, and double click on one to nominate a replacement icon. The replacement icon should then be shown next to the original icon.

Configuration

The theme editor gets many of it's resources from a resource directory. Should you wish, you can configure which directory it should retrieve resources from. To do so, select 'Preferences...' from the 'Edit' menu. This dialog also allows you to choose whether invisible items should be shown by default.

Command Line Arguments

If you pass a .qpkg file as a command line argument to it, it will load the specified file immediately. This, and '--help', are the only supported command line arguments at this time.

Limitations

The Qt Extended theme editor is only designed to facilitate common tasks in theme creation. The following things are not made easier by the theme editor, although the theme editor does try to help you in manually altering the text files.

  • List items
  • Plugin items
  • Widget items
  • Theme Translation
  • Adding xml files
