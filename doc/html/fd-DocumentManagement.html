Document Management
Functional Description

The Qtopia document system integrates with Digital Rights Management (DRM) and the Safe Execution Environment providing an enabling layer to enhance file-handling across all applications. Files are displayed by category rather than as a hierarchy which provides application-specific file displays. For example, if the user selects Mediaplayer, only the files able to be used by the Mediaplayer are displayed for selection.

Features

The Document Management architecture of Qtopia provides a system-wide approach to management of user-visible documents such as images and music files. This allows efficient navigation of documents, removable storage media, and complex document types such as Digital Rights Management (DRM) files. The SafeExecution system Document Server also relies on the Document Management system to control access to documents.

API

The document API framework provides the basis for the Document Management system and enables the browsing and launching of Digital Rights Management (DRM) files. An SQL database backs the framework allowing for flexible document management services.

The framework includes a scanning algorithm designed to run in the background and allow the user interface to continue to operate during document discovery. Qtopia applications in the launcher are also managed via the API.

Loading, saving and other file operations performed by Qtopia applications are channeled through the Document Management systems File Manager functionality. This allows management of DRM content for example, which must not be saved to external storage devices in plain-text format.

See the Document System API Documentation for further details.

Optimization of existing framework

Work has been done to reduce startup times and overhead involved with the new document system, including first run, through initial startup times, to general usage. Much of this related to the operation of the actual database engine on the device file-system, but also included SQL and other optimizations.

Cleanup of Public Facing API

Work has been done to clean up the public facing API of the Document System, to make it a complete and stable API. The intent is to enable Application Binary Compatibility through the 4.x Qtopia series, with as much planning for future features as possible. This is backed by a more efficient location storage and notification framework.

Model View Framework for launcher displays.

The previous LauncherView/LauncherIconView framework for displaying Applications/Documents/Games/Settings has been moved to a new Model/View framework, backed by QContentSet::model(). This provides better performance and "just-in-time" fetching of content data.

In line with the new QCopObject content channel notification architecture fewer messages are now passed for updates of the sets and thus the views.

A content filter expression system is integrated with the Document view/model. The system also handles the Qtopia application views (Applications, Games, Settings etc) and a visual notification of scanning for documents is provided.

Media Management

There is now provision for media management such as playlists, supporting MP3 ID3 tags, EXIF tags in JPEG photos etc.

A navigation view for documents tab, in cases where there are thousands of documents, a hierarchical (drill down) menu approach is used. Full-featured navigation by type and category is available.

This facility also allows selecting content on the SD Card.

Reworking of Image/!DocumentSelectors

Image and Document selector widgets available for use in other applications, now use the above optimized and full-featured framework.
