Accessories
Functional Description

The Accessory API is an extensible API that allows hardware accessories, such as headsets and batteries to be integrated into Qtopia. It provides a mechanism for managing the attributes of an accessory together with mechanisms for finding accessories that provide required functionality.

Features

The Accessory API is an extensible API that allows hardware accessories, such as headsets and batteries to be integrated into Qtopia. It consists of two classes:

  • QHardwareInterface - a base-class that provides the management of the accessory attributes. It is inherited to create the provider and accessory class for each accessory.
  • QHardwareManager - class used to find accessories that are available on the device. Methods are provided to search available accessories, and signals are emitted when accessories are added, removed or modified.

Some phones detect accessories with proprietary modem AT commands; e.g. AT*EACS in Ericsson phones. Other phones may use Linux kernel devices under /dev. Regardless of the mechanism, vendor-specific code inherits from the relevant subclass of QHardwareInterface and implements its specific functionality. The rest of the system can then detect the new accessory and act accordingly.
