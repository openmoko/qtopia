Detailed Description

The QSDPService class represents a bluetooth SDP record.

The QSDPService class encapsulates a Bluetooth SDP service record. The record is composed of Attributes, each attribute has a unique 16 bit id. The attributes can be of the following types:

  • quint8 - Represents a SDP UINT8 type.
  • qint8 - Represents a SDP INT8 type.
  • quint16 - Represents a SDP UINT16 type.
  • qint16 - Represents a SDP INT16 type.
  • quint32 - Represents a SDP UINT32 type.
  • qint32 - Represents a SDP INT32 type.
  • quint64 - Represents a SDP UINT64 type.
  • qint64 - Represents a SDP INT64 type.
  • quint128 - Represents a SDP UINT128 type.
  • qint128 - Represents a SDP INT128 type.
  • bool - Represents a SDP BOOL type.
  • QSDPAlternative - Represents a SDP Alternate type.
  • QSDPSequence - Represents a SDP Sequence type.
  • QSDPUUID - Represents a SDP UUID type.
  • QUrl - Represents a SDP URL type.
  • QString - Represents a SDP TEXT type.
  • QByteArray - Represents a SDP TEXT type that appears to contain binary data.

The attributes are stored as QVariants.

See also QVariant, QSDPSequence, and QSDPAlternative.

bool QSDPService::removeAttribute ( quint16 id )

int QSDPService::rfcommChannel ( const QSDPService & service )   [static]

For a family of services that work over the RFCOMM protocol, this method returns the RFCOMM channel the service is running on. The service parameter specifies the service record to search.

Returns the channel number on success, -1 if no channel number was found.
