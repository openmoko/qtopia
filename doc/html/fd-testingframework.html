QA: Testing Framework
Functional Description

Features

  • System Test Framework

System Test Framework

Qt Extended provides a framework (QtUiTest) for writing high-level system tests.

The functionality provided by the framework includes simulation of keypad and touchscreen input, and verification of expected behavior at the widget and application level. Verification takes place by textual or graphical snapshots of widgets. QtUiTest allows both manual and automated tests to be specified within the same test script, including some tests that may be partially automated and partially manual.

Test Scripts

System tests are written in a high-level scripting language, which is based on QtScript. Tests can be versioned in the source control system. Test scripts are composed of one or more test functions, each of which may have an associated set of test data. The test framework iterates over the test data set, calling the test function for each entry. The test data set is contained within the test script, but is separate from the test function. Common functionality may be specified in modules which can be referenced from within the test scripts.

Device and Implementation Independence

QtUiTest does not rely on internal data such as object names nor does it require a memory mapping mechanism to allocate widget elements. A test can therefore be completely written before the code even exists as long as there is an agreement about the Label names that in the User Interface. This also means that tests do not generally fail as a result of theme selection, display resolution, or minor UI implementation changes.

QtUiTest supports both keyboard based and touchscreen based devices out-of-the-box, and it does so in such a way that the script is transparent, i.e. can be run on both types of devices.

The tests can be run on a development workstation using the Qt virtual framebuffer, or on a target device.

Integration with Modem Simulator

QtUiTest is integrated with the Modem Simulator to enable testing of GSM related functionality. This includes simulation of incoming voice calls and SMS messages, including Class 0 (Flash) SMS.

Extension of QtUiTest Framework

QtUiTest can be extended by end-users for use with custom input methods and widgets via a plugin architecture. QtUiTest provides a plugin interface which allows plugins to be dynamically loaded to control interaction with widgets and input methods. When writing a new input method or custom input widget, it is possible to write a single new QtUiTest plugin and have all existing tests work with no modification.

Test Results

Test results are semantically and syntactically compatible with the results of tests written with Qt's QTestLib. This allows unit tests and automated system tests to be grouped together and run as a single set of tests.
