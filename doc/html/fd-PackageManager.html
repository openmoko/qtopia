Package Manager
Functional Description

Package Manager provides a number of functions relating to the management (installing, uninstalling, listing) of Qtopia .qpk packages. Where the Safe Execution Environment is enabled it further provides SXE related functionality, such as the viewing of security domains.

The Package Manager fetches the Package list from remote server via HTTP, or responds to a service request from the Web Browser; then displays package information, in particular a plain English description of what SXE security domains the package is requesting. Once a package has been selected, and downloaded the package manager will verify the SXE domains.

When packages are installed, if successful they are added to the launcher view for access and launching.

Features

Package Download

The Package Manager enables an end-user to

  • browse for games in the package manager UI
  • use a web browser which calls package manager as a service
  • select and download a package

Package Processing

Package processing functionality is automatic once the package is selected and downloaded. Functions are

  • verification, including checking that the browse-time md5sums match the downloaded package
  • adds to Document Management system so it can be launched via the Qtopia launcher

Package Processing SXE

SXE package processing functionality is available if the Safe Execution Environment is enabled. Functions are

  • verification, including checking that the browse-time domains match the downloaded package
  • domains not authorized by the user cause a "package corrupt" message
  • package install in a sand-boxed file structure and writes the MAC sandbox rules

Package Build and Publish

Scripts and documentation to enable and assist

  • a 3rd party package developer to build Qtopia 4.x SXE compatible software
  • build an SXE package from their sources
  • upload packages to a server
  • update the listing of packages available, including security domain profiles

Uninstall Packages

Packages may be securely uninstalled, removing them from all SXE definitions (if SXE is enabled) and from the launcher.

Re-enable Packages

If packages are disabled (in the case that SXE is installed) they may be temporarily re-enabled, for example in order to access valuable data stored by that application.
