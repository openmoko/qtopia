Detailed Description

The QPhoneProfileManager class allows applications to control phone profiles.

Qtopia's phone profiles are stored in the Trolltech/PhoneProfile configuration file. A device may have any number of integrator or user defined profiles. Each profile has a unique integer identifier which is used to refer to it by the system.

A profile can be manually activated through the QPhoneProfileManager::activateProfile() method. Alternatively a profile may be set to auto-activate at certain times, controlled through by QPhoneProfile::schedule(), or when a phone accessory is attached.

bool QPhoneProfileManager::activateProfile ( int profile )

Activate the profile id. If profile is -1 or the specified profile does not exist, a null (default values) profile will be activated.

bool QPhoneProfileManager::activateProfile ( const QPhoneProfile & profile )

This is an overloaded member function, provided for convenience.

Activate the profile. If profile has not been saved, the saved version with the same QPhoneProfile::id() will be activated.

This call is equivalent to activateProfile(profile.id()).

bool QPhoneProfileManager::planeMode () const

Return true if plane mode is active, false if not. Plane mode is active if either planeModeOverride() is true or QPhoneProfile::planeMode() for the active profile is true.

bool QPhoneProfileManager::planeModeAvailable () const

Returns true if plane mode is available, false if not.

void QPhoneProfileManager::planeModeChanged ( bool enabled )   [signal]

Emitted whenever the plane mode state changes. enabled will be set to true when plane mode is on, and false otherwise.

bool QPhoneProfileManager::planeModeOverride () const

Returns true if plane mode override is on, false otherwise. The plane mode override allows plane mode to be enabled, even if the active profile does not specify it as such.

See also setPlaneModeOverride().

Remove profile. If profile is a null profile, no action will be taken.
