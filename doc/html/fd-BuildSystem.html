Build System
Functional Description

The build system is the mechanism for building Qtopia in the various incarnations it has.

Features

  • Build Qt (host), Qtopia Core (target) and Qtopia.
  • Bypass qmake to give highly-parallel building without breaking dependencies.
  • Allow dependencies to be specified in the .pro files (easier maintenance of this information).
  • Allow dependency information to be extracted.
  • Run on Windows, Mac OS X and Linux (ie. handle cross-platform issues).
  • Create a directory that can be used to run Qtopia (ie. all the Qt/Qtopia files required should be present).
  • Build packages for third party applications (SXE depends on this).
  • Allow the use of GCC 4's symbol visibility feature.
