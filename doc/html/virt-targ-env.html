Introduction

The Qtopia Virtual Target Environment uses User Mode Linux (UML) to provide a development environment for Qtopia closely resembling target hardware. In particular, Safe Execution features may be enabled in the virtual system, leaving the host system as a regular development workstation.

The Virtual Target Environment allows testing to occur on a development machine with the entire device running as a single process. This, in turn, allows testing of Qtopia in different file system configurations or with different kernels while still being able to run and test the software without additional hardware. QVFb is used to simulate the display and input devices and benefits include being able to:

  • use QVFb to simulate the display and input devices
  • reset the state of the virtual target environment
  • perform entire device state diffs
  • limit device storage or RAM availability.
