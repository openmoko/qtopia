Building Qtopia Sync Agent

These instructions tell you how to build Qtopia Sync Agent, how to run a build and how to deploy Qtopia Sync Agent.

Requirements

Qtopia Sync Agent is known to build with Microsoft Visual C++ 6 (VC6), Microsoft Visual Studio .NET 2003 Standard Edition (VC7) and Microsoft Visual C++ Express 2005 (VC8).

Qtopia Sync Agent is known to build with the Platform SDK although the newer Windows SDK should also work.

You must have Outlook installed to build a fully-functional Qtopia Sync Agent. If you would like to target a different version of Outlook to the one you have installed you can place the files MSO.DLL and MSOUTL.OLB from the other version of Outlook into src/qtopiadesktop/dist.

Setup

Ensure that the prerequisites are met and any required environment variables are set. For each compiler a script will setup the environment. For example, VC8 installed to the default location has the script C:\Program Files\Microsoft Visual Studio 8\VC\bin\vcvars32.bat.

You should set QMAKESPEC to a suitable value.

CompilerQMAKESPEC
VC6win32-msvc
VC7win32-msvc.net
VC8win32-msvc2005

Build

    configure
    nmake
    nmake install

Execute Unit Tests

There is only minimal test coverage at this time, mostly aimed at the Outlook interface.

    cd src\qtopiadesktop
    nmake
    nmake test

Running

    dimage\qtopiasyncagent.exe

Deployment

You can build an installer for Qtopia Sync Agent. Please see src/qtopiadesktop/dist/README for details.
