SXE - Installer/Package Manager Overview

Qtopia provides a simplified software installation system in a system binary called packagemanger.

Note that earlier versions of Qtopia for PDA's provided a utility to install packages utilizing the ipkg system. There is no such facility currently in Qtopia. The SXE PackageManager is not a general purpose package installation system

Additionally to create safe execution images using the make install command during development and when readying a device image for shipping, on a device, the installer tool sxe_installer is provided.

The following features are provided:

  • fetch Package list from remote server via HTTP
  • display package information, in particular categories
  • ungzip, untar and install of qpk format packages
  • modifications to the two SXE policy rules tables
  • installation of run-time keys into binaries
  • provide uninstall capability.

Packages have dependencies on the version of Qtopia and device; packages which are not compatible will not be presented for download or be installable. There is no feature planned to have packages dependent on other packages, all packages will need to be self contained.

In future, packages that do not match the Qtopia version or device will be displayed (if desired) and be allowed to install. Notification will be given that packages these are unlikely to run correctly.

Note: Packages created by both non-SXE and SXE configured builds of Qtopia can be installed onto an SXE enabled device, provided a number of requirements are met.

Packges created by an SXE configured Qtopia however will not run or be available to install onto a non-SXE enabled device.

Applications which are untrusted run with the LIDS_SANDBOX attribute which means that only files specifically allowed may be accessed. Applications are by default installed in /home/Packages/<md5-sum> and have by default read-only access to all system libraries, and read-write access to their /home/Packages/<md5-sum> directory. Packages do not have access to the /tmp directory nor are they allowed to create QSettings objects by supplying an organization or application. To create temporary files or QSettings files, an application should use the Qtopia::sandboxDir() to obtain the package's directory; in the case of a QSettings file something like Qtopia::sandboxDir() + "/etc/mysettings.conf" should be supplied as a fileName parameter.

As a side note, packages cannot be quicklaunched applications, ensure that CONFIG+=no_quicklaunch is added into the application project(.pro) file.
