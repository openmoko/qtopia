Safe Execution Environment (SXE)

SXE Status

Important

SXE is released for trial use only, since at the current time full platform integration scripting and security auditing processes have yet to be completed.

For this reason SXE is controlled by the -sxe switch to the Qtopia configuration script. To enable SXE for conducting technology trials invoke the Qtopia configuration, and add -sxe to the command line.

SXE Overview

There is a growing demand among end-users for devices capable of downloading, installing and running third-party packages. This demand is driven in part by the availability of games and other applications conforming to the Java MIDlet specification .

Qtopia with the SXE is designed to enable these actions. It provides a level of security that allows for the device owner to have confidence in downloading and running natively compiled binaries without significant risk of compromising the operation of the device, or its data.

Unauthorized access to device services should not be possible. If anything unexpected is displayed on the screen the user should be able to exit the program by, for example, holding down the hangup button on the phone.

The SXE has three main features:

  • Public Key Infrastructure (PKI) for trusting packages:
  • Sand-boxing for packages not able to be trusted:
  • Application service level controls.
