Safe Execution Environment (SXE)

SXE Status

Important

SXE is scheduled to be released with Qtopia 4.3.1 which will allow Qtopia devices to safely run games. The following documentation is the residual of a prototype "System-wide Safe Execution Environment" that was placed in the 4.3.0 Beta for experimental purposes. The Safe Execution Environment that will be released will be considerably more lightweight and easy to use.

SXE Overview

There is a growing demand among end-users for devices capable of downloading, installing and running third-party packages. This demand is driven in part by the availability of games and other applications conforming to the Java MIDlet specification .

Qtopia with the SXE is designed to enable these actions. It provides a level of security that allows for the device owner to have confidence in downloading and running natively compiled binaries without significant risk of compromising the operation of the device, or its data.

SXE is a platform security solution. This means that it provides continuous policy based countermeasures designed to proactively prevent malicious or malfunctioning programs from harming assets on the device.

Platform security can be contrasted with:

  • perimeter solutions like firewalls, or systems relying on trust: these do not provide continuous security since once the program is through the perimeter gateway it is free to perform any action.
  • signature or trigger based solutions like anti-virus or change detection systems which react to signs of malicious activity before performing any actions

Platform security can be combined with perimeter and trigger type security to provide defence indepth. However platform security is a minimum requirement for a system which allows downloaded software to be run.

The SXE has several components:

  • Sand-boxing for packages not able to be trusted
  • Application service level controls.
  • General least-privileges and system-hardening policies
  • Public Key Infrastructure (PKI) for trusting packages (planned for a later release)
