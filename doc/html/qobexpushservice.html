Detailed Description

The QObexPushService class encapsulates an OBEX PUSH service.

The QObexPushService class can be used to provide OBEX PUSH services over Bluetooth, Infrared or any other OBEX capable transport. This class implements the Bluetooth Object Push Profile, and can also be used to implement the Infrared IrXfer service.

See also QObexServer.

Member Function Documentation

QObexPushService::QObexPushService ( QObexSocket * socket, QObject * parent = 0 )

Constructor for QObexPushService. The socket parameter specifies the OBEX socket to use. The socket is assumed to have been obtained from a QObexServer::nextPendingConnection call. The parent specifies the parent object.

Note that the QObexPushService takes ownership of the socket. When QObexPushService object is deleted, socket is deleted as well.

Destructor.

void QObexPushService::aboutToDelete ()   [signal]

This signal is emitted whenever an object is about to delete itself. This will only happen if the autoDelete property is set.

See also setAutoDelete() and autoDelete().

bool QObexPushService::acceptFile ( const QString & filename, const QString & mimetype, qint64 size )   [virtual protected]

Override this function to provide custom intelligence as to whether to accept an incoming file or not (e.g. popping a message box asking the user to accept, or using a file threshold, etc)

The filename holds the name of the file, mimetype holds the mimetype. The size parameter holds the size of the file.

The default implementation accepts all files unless no incoming directory is set.

bool QObexPushService::autoDelete () const

Returns true if autoDelete is enabled, returns false otherwise.

See also setAutoDelete().

QByteArray QObexPushService::businessCard () const   [virtual]

Clients should override this function to provide personal business card data in vCard format. This is required for Business Card exchange feature of the Bluetooth Object Push Profile. If an empty byte array is returned, the request for a business card will be rejected.

void QObexPushService::close ()

Stops the operation currently in progress, sets the state to Unconnected and if the autoDelete property is set to true, schedules deletion of the service.

void QObexPushService::done ( bool error )   [signal]

This signal is emitted whenever the client has disconnected from the service or the connection has been terminated. The error parameter reports whether an error occurred during processing.

QObex::Error QObexPushService::error () const

Returns the last error that has occurred.

void QObexPushService::getRequest ( const QString & filename, const QString & mimetype )   [signal]

This signal is emitted whenever a new get request comes in from the client. The only requests supported by the OBEX Push Service are for the Business Card. As such, the filename will parameter will always be a null string, and the mimetype will always be "text/x-vCard"

const QString & QObexPushService::incomingDirectory () const

Returns the incoming directory.

See also setIncomingDirectory().

void QObexPushService::progress ( qint64 completed, qint64 total )   [signal]

This signal is emitted reports the progress of the file send operation. The completed parameter reports how many bytes were sent, and total parameter reports the total number of bytes to send.

void QObexPushService::putRequest ( const QString & filename, const QString & mimetype )   [signal]

This signal is emitted whenever a new put request comes in from the client. The filename parameter contains the filename, and mimetype contains the mimetype of the request.

void QObexPushService::requestComplete ( bool error )   [signal]

This signal is emitted whenever a request has been completed. The error parameter holds whether an error has occurred during a request

void QObexPushService::setAutoDelete ( bool enable )

Sets the autoDelete property to enable. If autoDelete is enabled, then the object is automatically scheduled for deletion when the object next enters the Unconnected state.

Sets the incomding directory to dir. This directory will be used to place all files which are being received by the service.
