Environment Variables

Building Qtopia applications on a target system requires environment variables such as QTDIR to be set. If there is a requirement to use different versions of Qt, the environment variables point to the relevant library configurations.

Please see the Environment Variable Configuration for suitable environment variables to use.

Acceptable Directory Characters

Regular Expression Characters

Qtopia 1.9.x and Qtopia 2.0.0 - 2.1.0 have the following character requirements:

  • characters in any directory accessed by the Qtopia build system must not contain regular expression characters.
  • any special character or character sequence recognised by QRegExp may cause problems.
  • the '.' character is accepted as it matches itself.
  • regular expression characters in directory names must be avoided. These characters include, but are not limited to: + . ? * \ () [] ^ $ {}.

Note: A patch to remove this limitation exists for Qtopia 2.1.0. Qtopia 2.1.1 and above does not have this limitation.

Spaces

The Qtopia buildsystem does not support spaces in directory names. This limitation will not be fixed in the near future.

Please ensure the following:

  • no spaces appear in directory names accessed by the Qtopia build system
  • no spaces appear in paths leading to those directories.
