Introduction

The Qtopia 2.2.0 release is a minor release and includes the Qtopia Core, PDA, Phone and Refboard Editions.

Enhancements

  • Voice Over IP (VoIP) using SIP
  • SMTP authentication using SSL and TLS
  • 18bpp LCD support
  • Animated GUI
  • Live MP3 (SHOUTcast)
  • Transparency support
  • Dual Display support
  • openssl-0.9.7e added to 3rdparty libraries
  • Translations are available for US-English, GB-English, German, Japanese, Simplified Chinese, Portuguese/Brazilian, Spanish, French and Italian
  • Language-specific icon support
  • New packaging structure:
    • single package containing Qt, Qtopia and TMake
    • improved build process
    • improved start script
    • release and debug ARM builds included in SDK packages
    • improved support for singleexec builds
  • Major update of documentation:
    • improved documentation structure
    • functional specifications added to documentation
    • Qt documentation integrated into Qtopia documentation
    • documentation is edition-specific (Phone, PDA, etc)
    • significant re-write of large parts of the documentation

Optimizations

  • Cleaned up a range of source code warnings.
  • Optimized code for internationalization.
  • Improved performance for mediaplayer.
  • Improved startup time of qss, making it available almost immediately after system startup.
  • Moved timings for press and hold behavior as well as for multi-tap timeout into default buttons configuration file.

Known problems

  • Media filesystems limited to 8.3 filenames are problematic due to .desktop files.
  • MMS AMR recordings are cut short (with 0.5 sec) when playing back.
  • Typed numbers are deleted off the screen too quickly during a call.
  • QVFB doesn't gracefully handle a skin directory with missing contents.
  • Messages sometimes crashes when sending large attachments ( > 1.5MB ).
  • Messages does not gracefully handle out of storage situations.
  • Calling an unavailable VoIP contact does not display any message to say they are not connected.
  • On Phone edition, there is no way to set system volume besides using the mediaplayer and no way to set the input volume at all.
